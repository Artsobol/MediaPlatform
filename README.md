# Media Platform

Media Platform - микросервисный проект для работы с фото- и видеоконтентом. Идея платформы - дать основу для загрузки, хранения, обработки и выдачи медиафайлов через набор независимых сервисов.

Сейчас репозиторий находится на этапе базовой заготовки: добавлен корневой Maven-проект, который должен объединять будущие микросервисы в одну кодовую базу.

## Назначение

Проект задуман как медиа-платформа, в которой разные части системы отвечают за отдельные задачи:

- прием и загрузка фото/видео;
- хранение метаданных о медиафайлах;
- подготовка файлов к просмотру или дальнейшей обработке;
- выдача медиа пользователям или другим сервисам;
- дальнейшее расширение архитектуры без превращения системы в один большой монолит.

Такой подход удобен для учебного или pet-проекта, где можно постепенно развивать архитектуру: сначала собрать базовые сервисы, затем добавить очередь сообщений, файловое хранилище, авторизацию, API Gateway и наблюдаемость.

## Технологии

- Java 25
- Maven
- Spring Boot 4.0.6
- Микросервисная архитектура

## Планируемая архитектура

В дальнейшем проект может быть разделен на несколько сервисов:

- `media-service` - загрузка и управление медиафайлами;
- `metadata-service` - хранение описания, статусов и технических данных файлов;
- `processing-service` - обработка видео и изображений;
- `gateway-service` - единая точка входа для клиентских приложений;
- `auth-service` - аутентификация и авторизация пользователей.

Финальный набор сервисов может меняться по мере развития проекта.

## Запуск

Пока в проекте нет подключенных модулей с исполняемыми Spring Boot приложениями. Проверить текущую Maven-конфигурацию можно командой:

```bash
mvn test
```

Когда появятся отдельные сервисы, инструкции по локальному запуску, Docker Compose, переменным окружения и API-документации будут добавлены в этот README.

## Статус проекта

Проект находится в ранней стадии разработки. Сейчас это основа для будущей микросервисной медиа-платформы, а не завершенное приложение.
